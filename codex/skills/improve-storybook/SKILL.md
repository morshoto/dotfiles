# Improve Storybook

## Overview

Make Storybook feel like a first-class design system site by upgrading docs pages, standardizing story patterns, adding interaction tests, and keeping global configuration clean.

## Workflow

1. Scan `.storybook/main.ts` and `.storybook/preview.ts` to understand current addons, parameters, and decorators.
2. Pick target components and open their `*.stories.tsx` files.
3. Upgrade docs metadata and layout for each component story.
4. Convert to args-driven story exports with clean defaults.
5. Add `play` interaction tests where behavior is interactive or regression-prone.
6. Centralize shared configuration in `.storybook/preview.ts` and update addons in `.storybook/main.ts`.
7. Confirm Storybook still builds and stories render as expected.

## Docs Upgrades

- Add explicit `parameters.docs` blocks and component descriptions.
- Use `docs.page` to control the layout and include `Title`, `Description`, `Primary`, `Controls`, and `Stories`.
- Keep descriptions short and product-facing.

See `references/patterns.md` for docs block templates.

## Story Structure

- Use `StoryObj` with `args` for each story.
- Prefer one story per variant and keep names aligned with `args.variant`.
- Avoid custom render functions unless necessary.

See `references/patterns.md` for args-driven story examples.

## Interaction Tests

- Add `play` only when there is a user action or visual state transition to assert.
- Keep tests short and focused on user-visible outcomes.
- Use Testing Library helpers (`within`, `userEvent`).

See `references/patterns.md` for `play` examples.

## Global Configuration

- Centralize providers in `preview.ts` decorators.
- Set global `parameters` for layout and controls to keep stories clean.
- Add Storybook addons in `main.ts` and ensure packages are installed.

See `references/patterns.md` for configuration snippets.

## Done Criteria

- Docs pages show curated layout, not just autogenerated content.
- Stories are args-driven and composable.
- Interaction tests cover key click/hover/typing flows.
- Global decorators and addons are configured once, not repeated per story.
